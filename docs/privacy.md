# Privacy And Local Data

path to academia is local-first. It does not include telemetry, hosted accounts, analytics, or a
remote API client.

## What Is Stored

A workspace may contain:

- public or manually collected source records in `raw/`
- public fact tables in `tables/`
- audit notes in `audit/`
- private outreach status in `ui_state/outreach_status.csv`
- exported XLSX files generated by the user

Private notes, contact dates, application status, and next actions belong only in `ui_state/`.

## Local Server

The web UI binds to `127.0.0.1` by default. Do not expose it on a public interface unless the
workspace has been reviewed for private notes and unpublished source material.

## Publishing A Workspace

Before publishing a workspace or example:

- remove real private outreach notes
- remove private emails and contact history
- verify source URLs can be shared
- keep synthetic examples clearly labeled
- run `path-to-academia qa <workspace>`

## Security Reports

Report local file exposure, path traversal, accidental private-data mixing, or unsafe packaging
through the process in [../SECURITY.md](../SECURITY.md).
